Module 1

Operating Systems

History, functions, and types of operating systems.

Generations of Operating Systems

0th Generation
Mechanical Machines
Machines used wheels, cogs, and gears. Slow, unreliable. No OS existed.
1st Generation
Vacuum Tubes
Electrical designs replaced mechanical. Vacuum tubes — took up more space and heated quickly. No OS existed.
2nd Generation
Transistors
Vacuum tubes replaced by transistors. Had current leakage issues. Batch Operating System introduced.
3rd Generation
Integrated Circuits (IC)
Transistors replaced with silicon chips (IC). Portable, good Job Control Languages, supported Multiprogramming.
4th Generation
Large-Scale Integration
DOS, Linux, UNIX, and Windows introduced. Large-scale integration used.

Functions of the OS — PM DAU

Mnemonic: PM DAU

Processor Manager · Memory Storage Manager · Device Manager · Application Interface · User Interface

Processor Manager

Allocates enough processor time for each process/job so they run as efficiently as possible.

Memory Storage & Management

Ensures each process has enough memory and prevents processes from using each other's memory.

Device Manager

Manages I/O. Assigns high-priority blocks to drivers so hardware can be released for the next use quickly.

Application Interface (API)

Handles API actions so programmers don't need to know hardware control details.

User Interface

The part through which the user interacts. Windows & Macintosh use GUI; Unix uses shells.

Module 2

Boot Up

The startup sequence, key components, and I/O structures.

Key Definitions

Booting

The starting-up sequence of a computer when turned on — a series of operations performed in the computer.

BIOS

Basic Input/Output System. Pre-installed, low-level firmware on the motherboard. Initializes hardware on power-on. Contains hardware instructions to open up.

Bootable Device

Any storage medium the motherboard can detect and read to load the OS during startup. Examples: USB/Flash, Harddisk, SSD.

Boot Sector

Gateway to the OS — holds vital instructions needed for startup.

OS Kernel

The core program at the heart of a computer's operating system.

Boot-Up Sequence

1
The CPU executes an instruction in the BIOS containing the start-up program.
2
The program checks the devices connected to the computer.
3
BIOS looks for a bootable device.
4
Loads the Boot Sector and transfers execution into it.
5
Boot Sector loads and executes the OS Kernel to continue startup.

Interrupts

Interrupt

A signal from a device or program requiring the OS to stop and determine the next step. The computer either resumes the current program or begins running another.

Interrupt Handler: Included in the OS; prioritizes interrupts and saves them in a queue if more than one is waiting.

Trap / Exception

A software-generated interruption caused by an error or a specific user program request — a fault or access violation generated by the processor.

Hardware Interrupt

Signal to the processor comes from an external device.

Software Interrupt

User-programmed interrupt (or trap). Forces the program to jump to a well-known address based on the interrupt number.

I/O Structure

General-Purpose Computer System (GP-CS)

Consists of CPUs and multiple device controllers connected through a common bus.

Device Driver

Understands the device controller; presents a uniform interface to the rest of the OS. Typically one driver per device controller.

Device Controller

Responsible for a specific type of device. Maintains Local Buffer Storage (RAM) and a set of specific-purpose registers.

Interrupt-Driven I/O

1
Device Driver loads appropriate registers in the device controller.
2
Device Controller examines registers and transfers data from device to local buffer.
3
Interruption — once transfer is complete, driver is notified via interrupt.
4
Device Driver returns control to the OS. Note: Good for small data; high overhead for bulk data.

Direct Memory Access (DMA)

1
Device Driver sets up buffers, pointers, and counters for the I/O device.
2
Device Controller transfers an entire block of data directly to/from memory — no CPU intervention.
3
Interruption — only one interrupt per block tells the driver the operation is complete.
4
CPU is free to accomplish other tasks during the DMA transfer.

Module 3

Memory Management

Partitioning, allocation techniques, and deallocation strategies.

Single Contiguous Memory Scheme

Physical memory divided into two areas:
① Permanently allocated to the OS portion
② The User Process Area

Contiguous Memory Allocation Techniques

1. Static Partitioning

Pre-divided into fixed sizes. Partitions cannot be changed and can store only one process.

Example: 10 KB → (2kb | 3kb | 4kb | 1kb) — allocated based on the algorithm used.

2. Dynamic Partitioning

Uses a variable-size scheme; performs allocation dynamically. Partition size is determined by the size of the arriving process.

Allocation Techniques

TechniqueStrategy
First FitAllocate the first hole that is big enough.
Next FitSame as First Fit, but starts from the last allocated hole.
Best FitAllocate the smallest hole that is big enough.
Worst FitAllocate the largest hole.

Deallocation Techniques

Swapping

Process is temporarily swapped out of main memory to secondary, then swapped back in.

Example: Main Memory: P1 & P2; Secondary: P3–Pn. P1 goes for I/O wait → swaps out, P3 swaps in. After I/O, P1 returns and P3 swaps back.

Compaction

Shuffles memory so all free space comes together in one large block.

Problem: Must be done dynamically at execution time — not at static, assembly, or load time.
Cost: Expensive.

Example: 50 KB + 450 KB + 200 KB holes → combined into one 700 KB hole.

Module 5

Process Manager

Process management, PCB components, and CPU scheduling algorithms.

Key Definitions

Process Management

Includes creating and deleting processes, and providing mechanisms for processes to communicate and synchronize. Whenever there are several processes, they allocate the processor.

Process

Execution of a program — an execution unit where a program runs. Different tasks assigned to the system to perform.

Process States (5 Steps)

1
New — The process is created.
2
Ready — Waiting for the processor.
3
Running — The process is executing.
4
Waiting — Waiting for a resource (e.g., I/O or event).
5
Terminated — The process has finished executing.
Process Creation

After the OS loads, it creates an initial parent process. The parent generates several child processes.

Process Control Block (PCB)

Also called the control block. The data structure the OS maintains for every process.

#ComponentDescription
1Process IDUnique identifier of the process.
2Process StateCurrent state: new, ready, running, waiting, or terminated.
3Process PriorityNumerical value representing the process priority.
4Accounting InfoDescription/accounting of resources used by the process.
5Program CounterPointer to the next instruction to be executed.
6CPU RegistersHolds temporary information of the process when an interrupt halts it.
7PCB PointersAddress of the next PCB whose process state is ready.

CPU Scheduling

Preemptive Scheduling

Occurs when: (a) process switches running → ready (interrupt), or (b) waiting → ready (I/O completion).

Examples: A PWD takes your seat / Switching rooms because someone else needs it.

Algorithms: SRTF Priority Round Robin

Non-Preemptive Scheduling

Occurs when process switches running → waiting (I/O request). No chance of stopping even if other processes enter.

Examples: No one else can sit as long as you're sitting / Students wait for current class to finish.

Algorithms: FCFS SJF Priority Round Robin

Scheduling Algorithms

AlgorithmFull NameHow It Works
FCFSFirst Come, First ServeLike FIFO. The process that arrives first is executed first.
SJFShortest Job FirstJob with the smallest burst/duration time is scheduled first.
SRTFShortest Remaining Time FirstPreemptive version of SJF. Shortest remaining time runs next.
PriorityPriority SchedulingUrgent processes run first based on priority number.
RRRound RobinFixed quantum time per job. Process preempted after quantum. Cycles through queue. (Like alternating pizza slices.)

Enumeration

Must-Know Lists

Key enumerations and lists that commonly appear in exams.

5 Functions of the OS (PM DAU)

1
Processor Manager — Allocates the CPU among all processes.
2
Memory Storage Manager — Allocates memory; prevents processes from using each other's space.
3
Device Manager — Manages input/output devices via device drivers.
4
Application Interface — Allows application programs to communicate with the OS.
5
User Interface — Shell (Unix) or GUI (Windows/Mac) for user interaction.

5 Process States

1
New — The process is being created.
2
Ready — Waiting to be assigned to a processor.
3
Running — Instructions are being executed.
4
Waiting — Waiting for an event (e.g., I/O completion).
5
Terminated — The process has finished execution.

7 Components of the Process Control Block (PCB)

1
Process ID — Unique identifier of the process.
2
Process State — Current state (new, ready, running, waiting, terminated).
3
Process Priority — Numerical value representing urgency.
4
Accounting Info — Resources used by the process.
5
Program Counter — Pointer to the next instruction to execute.
6
CPU Registers — Holds temporary data when an interrupt halts the process.
7
PCB Pointers — Address of the next PCB in ready state.

4 Memory Allocation Techniques

1
First Fit — Allocate the first hole big enough.
2
Next Fit — Like First Fit but starts from the last allocated hole.
3
Best Fit — Allocate the smallest hole that is sufficient.
4
Worst Fit — Allocate the largest available hole.

5 CPU Scheduling Algorithms

1
FCFS — First Come, First Serve. Non-preemptive; like FIFO.
2
SJF — Shortest Job First. Non-preemptive; smallest burst time runs first.
3
SRTF — Shortest Remaining Time First. Preemptive version of SJF.
4
Priority Scheduling — Highest-priority process runs first. Can be preemptive or non-preemptive.
5
Round Robin (RR) — Preemptive; each process gets a fixed quantum time, then cycles to the next.

7 Levels of Storage Hierarchy (Top → Bottom)

1
Registers — Fastest, smallest, most expensive. Volatile.
2
Cache — Very fast. Volatile.
3
Main Memory (RAM) — Fast. Volatile.
4
Electronic Disk (SSD) — Medium speed. Non-volatile.
5
Magnetic Disk — Slower. Non-volatile.
6
Optical Disk — Slow. Non-volatile.
7
Magnetic Tapes — Slowest, largest, cheapest per bit. Non-volatile.

4 OS Generations

1
1st Generation — Vacuum tubes. No OS yet. Replaced mechanical machines.
2
2nd Generation — Transistors (replaced vacuum tubes). Had current leakage issues.
3
3rd Generation — Silicon chips / Integrated Circuits (IC). Introduced multiprogramming.
4
4th Generation — Large-scale integration. Introduced DOS, Linux, UNIX, and Windows.

4 Types of Operating Systems

1
Real-Time OS — Controls machinery, scientific instruments, industrial systems (e.g., robotics, avionics).
2
Multiuser OS — Multiple users simultaneously share resources (e.g., Windows Server, UNIX, Linux).
3
Single User, Single Task — One user, one task at a time (e.g., PDAs, early mobile OS).
4
Single User, Multitask — One user, multiple tasks simultaneously (e.g., modern desktop Windows/Mac).

Boot-Up Sequence (5 Steps)

1
CPU executes the BIOS firmware stored on the motherboard.
2
BIOS checks and inventories all hardware devices (POST).
3
BIOS locates a bootable device (hard drive, USB, etc.).
4
Boot Sector is loaded from the bootable device into memory.
5
Kernel is loaded by the Boot Sector — the OS is now running.
